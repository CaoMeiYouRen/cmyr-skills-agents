# 交叉核查案例：AI 一人企业报告

> 日期：2026-06-11 | 来源：草梅友仁 的会话

## 背景

一份关于"AI + 一人企业"的调研报告引用了 smartcity.team 的 OPC 深度研究报告中的多个案例数据。用户要求对报告中所有声明进行交叉核查。

## 发现的问题

### 无法验证的数据

| 声明 | 原始来源 | 核查结果 |
|------|---------|---------|
| "美国 2980 万独立创业者，$1.7 万亿 GDP" | MetaEra 文章（非官方统计） | ❌ 未找到独立验证来源 → 删除 |
| "澳大利亚华人创业者月流水 $100K" | smartcity.team 报告（第三方转述） | ⚠️ 仅出现在聚合报告中，未找到创始人一手来源 → 降级标注 |
| "龙骨动画苏魁收入近万元" | smartcity.team 报告（第三方转述） | ⚠️ 同上 → 降级标注 |

### 已验证的数据（多源交叉）

| 声明 | 验证来源 |
|------|---------|
| 中国 OPC 1600 万+（2025.06） | 新华社、BBC、腾讯新闻、东方财富 |
| 2025H1 新注册 286 万户，+47% | 同上 |
| HACR ¥1→¥72 | 东方财富、新浪财经、36氪、人民网 |
| Avneesh Kumar 6产品/$180-250 | 一手博客 avneesh.com |
| 陈少波 6平台分发 | 一手文章 juejin.cn |

## 教训

**第三方聚合报告（smartcity.team 类）是高风险信息来源。** 它们可能引用几十个案例，但多数是二手转述，搜不到原始出处。数值声明（收入、成本、用户量）若仅出现在聚合报告中且无一手来源链接，不可作为事实写入最终报告。

## 核查方法论

1. 提取所有数值/案例声明
2. 逐条搜索原始出处（一手博客、官方页面、原始采访）
3. 对第三方转述数据要求至少 2 个独立验证来源
4. 搜不到一手来源的降级或删除
5. 在核查报告中标注每条声明的置信度（⭐⭐⭐ / ⭐⭐ / ⭐）
