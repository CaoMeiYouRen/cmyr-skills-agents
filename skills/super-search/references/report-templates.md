# Report Templates

## 模板选择指南

根据用户意图选择报告模板，不要在全文用占位符：

| 主题特征 | 推荐模板 | 示例主题 |
|----------|----------|----------|
| 多产品/多方案价格、功能对比 | **对比型报告模板** | LLM 定价对比、域名注册商对比、VPS 对比 |
| 单一问题/错误的根因排查 | **诊断型报告模板** | VS Code EMFILE 错误、API 限流问题 |
| 某个市场/赛道的生态调研 | **对比型报告模板**（按平台/玩家分组） | LLM 聚合网关生态、支付平台生态 |
| 简单事实查证（quick 模式） | **快速摘要模板** | "XX 工具是否免费""YY 事件最新进展" |

IRON RULE: 报告中每条事实声明必须有可追溯的 URL。不允许出现 `_（请手动填写）_` 类占位符。

---

## 数据标注规范

所有报告中统一使用以下标注：

| 标注 | 含义 | 使用场景 |
|------|------|----------|
| `✅ 官方核实` | 从官方定价页/文档直接抓取并确认 | 官方页面可直接获取的数据 |
| `✅ 第三方交叉验证` | 2+ 个独立第三方来源一致确认 | 官方页面无法抓取但多方一致 |
| `⚠️ 第三方单源` | 仅有 1 个第三方来源 | JS 渲染定价页 + 仅1个第三方汇总 |
| `⚠️ 推广性质` | 来源有明显商业推广嫌疑 | 软文、广告、厂商自述 |
| `❓ 待验证` | 孤立声明，无第二来源 | 社区帖子、个人博客中的声明 |
| `❌ 无法核实` | 所有抓取方式均失败 | 反爬/JS 渲染/403 |
| `📝 行业已知` | 基于行业常识，非本次搜索直接核实 | 知名服务的公开定价 |

## 四、深度研究追加模板（deep 模式）

在对比型/诊断型报告基础上追加以下 sections：

### 来源立场分析
```markdown
## 来源立场分布
| 立场 | 来源数 | 代表来源 |
|------|--------|----------|
| 支持 | 5 | [url1](url1), [url2](url2) |
| 中立 | 3 | [url3](url3) |
| 反对/批评 | 2 | [url4](url4) |
```

### 数据一致性分析
```markdown
## 数据一致性
| 数据点 | 来源数 | 数据范围 | 一致性 |
|--------|--------|----------|--------|
| 增长率 | 3 | 12% - 15% | 基本一致 |
| 定价 | 5 | $10 - $15 | 一致 |
```

### 信息来源图谱
```markdown
## 来源关系
| 来源类型 | 数量 | 代表性来源 |
|----------|------|-----------|
| 官方来源 | {count} | [url1](url1) |
| 学术来源 | {count} | [url2](url2) |
| 权威媒体 | {count} | [url3](url3) |
| 行业分析 | {count} | [url4](url4) |
| 社区/UGC | {count} | [url5](url5) |
```

### 用户画像推荐矩阵（适用于对比型 deep 报告）

```markdown
|----------|----------|------|
```

## 输出路径规则

默认输出路径：`{cwd}/research-output/{date}-{topic-slug}.md`

用户可通过参数覆盖：
```bash
node dist/report.mjs --output ./custom/path/report.md
```

## 报告质量检查清单

交付前逐项确认：

- [ ] 报告中每条事实声明都有可追溯的 URL
- [ ] 低质量来源（综合评分 < 30）已排除并在「已舍弃的来源」中列出
- [ ] 所有引用 URL 已通过来源可信度核查（高风险 TLD 如 .cc/.xyz/.top 已标注/排除）
- [ ] 对比表/关键数据优先链向官方页面而非第三方文章
- [ ] 第三方转载数据已满足 2 源交叉比对要求
- [ ] 矛盾点已通过官方来源核查并在交叉验证中标注结论
- [ ] 孤立声明已在交叉验证中明确标注「待验证」且置信度不超过 medium
- [ ] 涉及定价/费率/佣金的数据有 2+ 独立来源确认
- [ ] 时效性审查：所有数据来源日期在可接受范围内（6 个月内为佳）
- [ ] 无 `_（请手动填写）_` 类占位符
- [ ] 代理/推广站内容未被当作官方信息引用
- [ ] 推广性质的内容已在来源表中标注「⚠️ 推广性质」
- [ ] 报告最后一行包含 `*Powered by CaoMeiYouRen*`
